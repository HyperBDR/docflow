# DocFlow Recorder

## 构建与安装

```bash
npm install
npm run build
```

在 Chrome 或 Edge 打开扩展管理页，启用开发者模式，选择“加载已解压的扩展”，加载 `extension/dist`。

生成正式 ZIP、准备隐私与权限说明，以及发布到 Chrome Web Store / Microsoft Edge Add-ons 的完整流程见 [PUBLISHING.md](PUBLISHING.md)。

## 使用

1. 在插件中点击连接账号，插件会打开 DocFlow 专用连接页。未登录时先完成登录，随后页面会自动检测扩展并通过一次性授权完成连接，不需要点击额外按钮，也不需要填写 API 地址或配对码。
2. 直接切换到需要录制的业务页面，在插件主页选择 **Record Interactive Demo**，无需提前创建演示。页面弹框可选择推荐的 **HTML Cloning** 或 **Screenshot**，并在底部选择保存到哪个团队/个人空间以及是否启用 AI。
   弹框会同时检查浏览器窗口尺寸和当前窗口的标签页数量：尺寸偏差较大时可一键调整到 1056×660，标签页过多时可关闭当前页之外的非固定标签页。这些提示不会阻止用户直接开始录制；尺寸会实时重新检测，调整后再次还原窗口时警告会重新出现。
3. 每种模式第一次使用时会展示三步录制教学；完成后将不再重复显示，可在插件设置中选择“下次重新显示录制教学”。点击 `Get Started` 时插件才会在选中的空间中自动创建草稿，取消设置不会留下空演示。
4. 如果要给已有演示补充步骤，可在演示编辑页点击“继续录制”；这个追加目标只对下一次录制生效，不会影响以后新建的录制。
5. HTML Cloning 录制时移动鼠标会高亮候选元素；点击后等待短暂上传完成，再选择下一个元素。
6. 页面左下角控制条可拖动，支持 Pause / Resume、查看步骤数量、无热点手动截图和 Stop Recording。录制期间切换到任何普通 HTTP/HTTPS 标签页时，该标签页会自动加入同一录制会话并显示录制状态；新打开的标签页和窗口同样会自动加入。停止后编辑器会自动打开。

每次选择元素后，扩展只在上传截图、经过清洗的 DOM 快照、目标元素选择器和滚动位置时短暂锁定页面，服务端确认入库后立即允许继续选择。如果本次录制启用了 AI，API 会自动把步骤提交给后台 Worker，标题、说明和热点提示生成不会阻塞后续录制。手动截图会保存当前 HTML 与截图但不创建热点。控制条、模式弹窗和教学弹窗不会进入截图或 DOM 快照。HTML 模式会尝试通过扩展权限内联跨域样式表、字体和图片；访问受限的资源仍使用截图作为视觉回退。人工修改过的字段不会被后续 AI 结果覆盖。

构建会读取项目根目录 `.env` 中的 `DOCFLOW_PUBLIC_BASE_URL` 与 `DOCFLOW_WEB_ORIGIN`，也可用 `DOCFLOW_EXTENSION_API_URL`、`DOCFLOW_EXTENSION_WEB_URL` 覆盖。更新扩展代码并重新执行 `npm run build` 后，需要回到扩展管理页点击“重新加载”；插件会为已经打开的普通 HTTP/HTTPS 页面自动补充录制脚本，无需再手动刷新目标页面。

浏览器内部页面、扩展商店及部分跨域 iframe 受浏览器安全策略限制，不能录制；这些步骤可在编辑器中手动上传截图。
